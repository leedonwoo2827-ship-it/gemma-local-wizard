# SQLD 모의고사 제작 최종 시나리오 (회사 PC 기준)

Auto Gemma Starter 를 엔진으로 "스캔 → 전사 → 출제 → 요약 → 음성/슬라이드 → 렌더링" 을 진행하는 실전 순서.

> 준비: 회사 PC 는 사양이 좋으므로 **gemma3:4b 이상**(가능하면 12b) 설치 권장. 이미지 OCR 은 4b+ 필수(1b 은 텍스트 전용).

---

## 0. 설치 & 저장 위치 (최초 1회)
1. `run.bat` 실행 → 마법사 열림.
2. **⚙ 설정 → 모델 저장 위치 → "추천 위치"**(여유 큰 드라이브, 예 `D:\AutoGemma\models`) → 저장 및 적용.  ← 모델 받기 **전에** 필수
3. **2. Ollama** → "Ollama 설치".
4. **3. 모델 관리** → `gemma3:4b`(또는 12b) 설치.

---

## 1단계 — 스캔 이미지 OCR·전사 (봇 ①)
- 채팅 창 → 봇 **① SQLD OCR·문제 전사**, 모델 `gemma3:4b+`.
- `이미지` 체크 → 시험지 사진 첨부 → "이 시험지를 전사해줘".
- 결과(문항별 텍스트)를 검수·수정하여 **문제별 .md 파일**로 저장.
  - 권장: **회차별 1개 파일**(예 `1회.md` … `10회.md`) → 낱개 500개보다 관리·출처 추적이 쉬움.
- 반복하여 10회 × 50문 = 500문항 확보.

## 2단계 — 지식 도서관에 적재 (RAG)
- 좌측 **지식 도서관(RAG)** 탭 → **"+ 폴더 추가"** 로 md 폴더를 통째로 업로드(하위 폴더까지 자동 수집).
  - 또는 **"+ 파일 추가"** 로 여러 파일 다중 선택.
- 진행바로 `N/총` 확인, 완료 시 "추가/실패/조각 수" 표시.

## 3단계 — 20회분(50문항) 출제 (봇 ②)
- 봇 **② SQLD 모의고사 출제**, `지식 도서관 사용` 체크.
- ⚠️ **한 번에 500개를 프롬프트에 넣지 말 것**(컨텍스트 초과). 아래처럼 나눠 진행:
  - "**1회분 50문항** 만들어줘. 데이터 모델링 20% / SQL 80% 비율로." → 회차별로 20번 반복.
  - 또는 주제별("SELECT·조인 위주 25문항", "DDL·제약조건 25문항")로 쪼개 생성.
- 각 회차 결과를 `모의고사_1회.md` … `20회.md` 로 저장.
- (선택) 고급 설정에서 `num_ctx` 를 16K~32K 로 올리면 한 번에 더 많이 참고 가능(속도·메모리 부담↑).

## 4단계 — 해설 요약이론 (봇 ③)
- 봇 **③ SQLD 해설 요약이론**, `지식 도서관 사용` 체크(해설 문서 근거).
- "업로드한 해설을 주제별로 요약이론 만들어줘. ⭐빈출 표시." → `요약이론.md` 저장.

## 5단계 — AI 음성 대본 (봇 ④)
- 봇 **④ AI 음성 대본** → 요약이론/해설을 붙여넣고 "나레이션 대본으로 바꿔줘".
- 결과 `대본.md` 저장 (SQL→에스큐엘 등 읽기 변환, [쉼]·(슬라이드 N) 포함).

## 6단계 — PPTX 슬라이드 기획 (봇 ⑤)
- 봇 **⑤ PPTX 슬라이드 기획** → 요약/대본 붙여넣고 "슬라이드로 정리해줘".
- 결과 `슬라이드.md` 저장 (슬라이드별 제목·불릿·발표자 노트).

## 7단계 — 파일 생성 & 렌더링 (앱 밖, 도구 연계)
LLM 산출물(md)을 실제 파일로 만드는 단계 — 별도 도구 사용:
- **PPTX**: `슬라이드.md` → `pptx_writer` 로 .pptx 생성.
- **책(HWPX)**: 봇 **⑥ HWPX 교재 원고** 로 원고 작성 → `hwpx_writer` 로 .hwpx 생성.
- **영상**: 대본 → TTS 음성 + 슬라이드 이미지 → ffmpeg 등으로 합성·렌더링.

---

## 핵심 주의사항
1. **저장 위치 먼저**: 모델 받기 전에 ⚙ 설정에서 D: 지정 (C: 용량 절약).
2. **컨텍스트 한계**: 500문항 통째 입력 불가 → RAG 적재 + 회차/주제 단위 반복.
3. **단계마다 `+ 새 대화`**: 봇을 바꾸면 이전 맥락이 섞일 수 있음 → 단계별 새 대화 권장.
4. **모델 성능**: 4b/12b 일수록 출제·요약 품질↑. OCR·이미지는 4b+ 필수.
5. **결과 저장**: 각 단계 결과는 `내보내기`(md/txt) 또는 복사로 저장해 다음 도구로 전달.
